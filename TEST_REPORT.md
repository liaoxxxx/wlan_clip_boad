## 已知问题

### 1. Windows 自动输入功能限制

**问题描述**: 
- 逐字符输入功能（sendKeys）暂时禁用
- 原因：win32 5.15.0 包的 INPUT 结构体 API 变化

**当前状态**:
- ✅ 剪贴板写入功能正常
- ⚠️  键盘模拟功能需要等待 win32 包更新或寻找替代方案

**临时解决方案**:
- 使用"粘贴方式"模式
- 文本会写入剪贴板，用户手动按 Ctrl+V 粘贴

**后续改进**:
- 研究 win32 包的正确用法
- 或考虑使用其他键盘模拟库
- 或使用 Windows API 的直接 FFI 调用

### 2. Kotlin 编译器缓存问题（已解决）

**问题描述**: 
- 构建过程中出现 Kotlin daemon 编译错误
- 多个根目录路径冲突导致增量编译失败

**解决方案**:
- 执行 `gradlew.bat --stop` 停止所有 Gradle daemon
- 清理项目缓存后重新构建

**验证结果**:
