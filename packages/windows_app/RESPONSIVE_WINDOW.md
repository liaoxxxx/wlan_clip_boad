# 窗口纵向尺寸自适应功能说明

## ✨ 功能概述

Windows 应用现已支持窗口纵向尺寸自适应功能。当用户垂直调整（缩小）窗口高度时，界面会自动隐藏非核心组件，仅保留并完整显示"接收到的文本"展示组件。

## 📏 响应式布局模式

应用根据窗口高度自动切换到不同的显示模式：

### 1. 完整模式 (Window Height ≥ 450px)
**显示所有组件：**
- ✅ AppBar（标题栏）
- ✅ 服务器状态卡片（含 IP 地址）
- ✅ 接收的文本区域
- ✅ 输入模式选择
- ✅ 悬浮窗模式设置
- ✅ 使用说明提示
- ✅ 连接日志区域

### 2. 紧凑模式 (300px ≤ Window Height < 450px)
**隐藏部分次要组件：**
- ✅ AppBar（标题栏）
- ✅ 服务器状态卡片（不含 IP 地址详情）
- ✅ 接收的文本区域
- ✅ 输入模式选择（不含粘贴方式选项）
- ❌ 悬浮窗模式设置
- ❌ 使用说明提示
- ❌ 连接日志区域

### 3. 最小模式 (Window Height < 300px)
**仅显示核心组件：**
- ❌ AppBar（标题栏隐藏）
- ❌ 服务器状态卡片
- ✅ 接收的文本区域（简化版，只显示图标）
- ❌ 输入模式选择
- ❌ 悬浮窗模式设置
- ❌ 使用说明提示
- ❌ 连接日志区域

## 🎯 核心特性

### 1. 智能组件隐藏
- 自动检测窗口高度变化（每 200ms 检查一次）
- 根据阈值自动切换显示模式
- 平滑过渡，无闪烁

### 2. 文本区域优化
- **始终可见**：无论窗口多小，文本区域始终显示
- **字体自适应**：最小模式下字体缩小到 12px
- **间距优化**：最小模式下边距减小到 8px
- **Flex 比例调整**：根据模式调整文本区域的 flex 比例

### 3. 最小窗口限制
- 最小窗口尺寸：380 x 200 px
- 确保文本内容依然清晰可见
- 防止布局崩坏

## 🔧 技术实现

### 关键参数
```dart
static const double _compactThreshold = 450.0;  // 紧凑模式阈值
static const double _minimalThreshold = 300.0;  // 最小模式阈值
static const double _minWindowHeight = 200.0;   // 最小窗口高度
```

### 窗口高度监听
```dart
// 每 200ms 检查一次窗口尺寸
_windowSizeTimer = Timer.periodic(const Duration(milliseconds: 200), (_) {
  if (mounted) {
    _updateWindowHeight();
  }
});
```

### 条件渲染
使用 Dart 的条件集合操作符实现组件的动态显示/隐藏：
```dart
if (!_isMinimalMode) ...[
  // 组件列表
],
```

   - 慢慢向下拖动窗口底部边框

     - 粘贴方式选项消失
     - 悬浮窗设置消失
     - 使用说明消失

4. **测试最小模式**
   - 继续缩小窗口高度
   - 当高度小于 300px 时：
     - 标题栏（AppBar）消失
     - 服务器状态卡片消失
     - 文本区域标题简化为 "📝"
     - 复制按钮消失
     - 文本字体变小

5. **测试最小限制**
   - 尝试继续缩小窗口
   - 确认窗口不能小于 200px 高度
   - 确认文本内容依然清晰可读

### 预期行为

✅ **正常行为：**
- 窗口缩小时，组件逐渐隐藏
- 文本区域始终保持可见
- 布局不崩坏，无重叠

## 💡 设计考虑

### 为什么这样设计？

1. **用户体验优先**
   - 核心功能（文本显示）始终可用
   - 次要功能在空间不足时自动隐藏
   - 避免滚动条过多，保持简洁

2. **性能优化**
   - 减少不必要的组件渲染
   - 降低内存占用
   - 提高小窗口下的响应速度

3. **实际场景**
   - 用户可能希望将应用放在屏幕角落
   - 只需查看接收的文本
   - 不需要复杂的控制选项

### 阈值选择依据

- **450px（紧凑模式）**：足够显示状态信息和基本控制
- **300px（最小模式）**：刚好能舒适地显示文本内容
- **200px（最小限制）**：确保文本可读性的绝对下限

## ⚠️ 注意事项

1. **性能影响**
   - 定时器每 200ms 执行一次，性能开销极小
   - 仅在窗口大小变化时触发 setState

2. **兼容性**
   - 仅适用于 Windows 平台
   - 使用 bitsdojo_window API

3. **托盘功能**
   - 与托盘功能完全兼容
   - 窗口隐藏到托盘不受影响
