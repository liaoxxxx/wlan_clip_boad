# Windows 托盘功能使用说明

## ✨ 功能特性

Windows 版本现已支持最小化隐藏到系统托盘的功能，包括：

1. **点击关闭按钮**：窗口不会真正关闭，而是隐藏到系统托盘
2. **托盘菜单**：右键点击托盘图标可显示菜单
   - 显示窗口
   - 隐藏窗口
   - 退出应用
3. **后台运行**：即使窗口隐藏，服务仍在后台运行，继续接收 Android 端的剪贴板同步

## 🚀 使用方法

### 启动应用

```bash
cd packages/windows_app
flutter run -d windows
```

### 隐藏到托盘

- **方法1**：点击窗口右上角的关闭按钮（X）
- **方法2**：通过托盘菜单选择"隐藏窗口"

### 从托盘恢复

1. 在系统托盘中找到 ClipSync WiFi 图标
2. 右键点击图标
3. 选择"显示窗口"

### 完全退出应用

1. 右键点击托盘图标
2. 选择"退出"

## 📝 注意事项

### 托盘图标

默认情况下，如果找不到自定义图标文件，系统会使用默认的 Windows 应用图标。

如需使用自定义图标：

1. 准备一个 `.ico` 格式的图标文件（建议包含 16x16, 32x32, 48x48 尺寸）
2. 将文件命名为 `tray_icon.ico`
3. 放置在 `packages/windows_app/assets/icons/` 目录下

### 依赖包

本功能使用了以下 Flutter 包：

- `system_tray`: ^2.0.3 - 系统托盘支持
- `bitsdojo_window`: ^0.1.6 - 窗口自定义和控制

这些依赖已添加到 `pubspec.yaml` 中。

## 🔧 技术实现

### 核心文件

1. **lib/windows/tray_manager.dart**
   - 系统托盘管理器
   - 处理托盘图标初始化
   - 管理托盘菜单

2. **lib/windows/windows_server.dart**
   - 集成托盘功能
   - 拦截窗口关闭事件
   - 实现显示/隐藏逻辑

3. **lib/main.dart**
   - 配置 bitsdojo_window
   - 设置窗口行为

### 工作流程

```
用户点击关闭按钮
    ↓
bitsdojo_window 拦截关闭事件
    ↓
调用 _hideWindow() 方法
    ↓
窗口隐藏，应用继续在后台运行
    ↓
托盘图标保持可见
    ↓
用户可通过托盘菜单恢复窗口或退出应用
```

## ⚠️ 常见问题

### 1. 托盘图标不显示

**可能原因**：
- 图标文件不存在或路径错误
- system_tray 初始化失败

**解决方法**：
- 检查控制台输出，查看是否有错误信息
- 确保 `assets/icons/tray_icon.ico` 文件存在（可选）
- 即使没有自定义图标，应用也会使用默认图标正常运行

### 2. 点击关闭按钮后应用完全退出

**可能原因**：
- bitsdojo_window 未正确配置

**解决方法**：
- 确认 `main.dart` 中已正确调用 `doWhenWindowReady()`
- 确认已添加 `bitsdojo_window` 依赖

### 3. 托盘菜单无法弹出

**可能原因**：
- 托盘初始化失败
- Windows 权限问题

**解决方法**：
- 以管理员身份运行应用
- 检查 Windows 通知区域设置，确保应用图标未被隐藏
