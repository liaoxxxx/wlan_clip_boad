### 3. 功能特性

#### 窗口关闭行为
- 点击窗口关闭按钮（X）→ 隐藏到托盘（不退出应用）
- 应用继续在后台运行，WebSocket 服务保持活跃
- Android 端仍可正常发送剪贴板内容

#### 托盘菜单
右键点击托盘图标显示菜单：
- **显示窗口**：恢复应用窗口
- **隐藏窗口**：隐藏应用窗口到托盘
- **退出**：完全退出应用

#### 托盘点击事件
- **左键点击**：显示窗口
- **右键点击**：弹出菜单（Windows 自动处理）

### 4. 技术实现细节

#### 窗口拦截机制
使用 `WillPopScope` widget 拦截返回键/关闭按钮事件：
```dart
WillPopScope(
  onWillPop: () async {
    _hideWindow();
    return false; // 阻止默认关闭行为
  },
  child: MaterialApp(...),
)
```

#### 托盘初始化流程
1. 应用启动时调用 `_initTray()`
2. 初始化 SystemTray 实例
3. 设置托盘图标和提示文本
4. 构建上下文菜单
5. 注册事件处理器

#### 窗口显示/隐藏
- 使用 `appWindow.show()` 显示窗口
- 使用 `appWindow.hide()` 隐藏窗口
- 通过 `_isWindowHidden` 状态跟踪窗口可见性

## 📝 使用说明

### 启动应用
```bash
cd packages/windows_app
flutter run -d windows
```

### 隐藏到托盘
- 点击窗口右上角的关闭按钮（X）

### 从托盘恢复
- 左键点击托盘图标，或
- 右键点击托盘图标 → 选择"显示窗口"

### 完全退出
- 右键点击托盘图标 → 选择"退出"

## ⚠️ 注意事项

### 托盘图标
- 如果 `assets/icons/tray_icon.ico` 不存在，系统会使用默认图标
- 应用仍可正常运行，只是托盘图标为默认样式
- 建议添加自定义图标以获得更好的用户体验

### 已知限制
1. **WillPopScope 可能无法拦截所有关闭场景**
   - 某些系统级别的关闭操作可能仍会直接退出
   - 如需更可靠的拦截，可能需要使用原生 Windows API

2. **托盘图标在开发模式下可能不显示**
   - 这是正常现象，打包后的 release 版本会正常显示

3. **首次运行可能需要防火墙权限**
   - WebSocket 服务需要网络访问权限
