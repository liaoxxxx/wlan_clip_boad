# 📌 悬浮窗功能说明

## ✨ 新功能：悬浮窗模式

### 功能描述
Windows 应用现在支持**悬浮窗模式**（Always on Top），启用后窗口会始终保持在屏幕最前面，不会被其他窗口遮挡。

### 使用场景
1. **实时监控** - 随时查看从 Android 端接收的文本
2. **多任务工作** - 在其他应用中工作时，仍能看见同步的文本
3. **演示展示** - 向他人展示剪贴板同步功能
4. **快速参考** - 将重要信息保持在可见区域

---

## 🎯 如何启用

### 步骤 1: 打开应用
启动 Windows 应用 `clip_sync_wifi.exe`

### 步骤 2: 找到窗口设置
在应用界面中找到 **"📌 窗口设置"** 区域（紫色边框）

### 步骤 3: 开启悬浮窗
点击 **"悬浮窗模式"** 开关
- ✅ 开启：窗口始终在最前面
- ❌ 关闭：窗口正常层级

### 步骤 4: 验证效果
尝试打开其他窗口，Clip Sync WiFi 窗口应该仍然可见

## 🔧 技术实现

### Win32 API
使用 Windows API 实现窗口置顶：

```cpp
// 设置窗口为 TOPMOST
SetWindowPos(
    hwnd,                    // 窗口句柄
    HWND_TOPMOST,           // 置于最前
    0, 0, 0, 0,             // 不改变位置和大小
    SWP_NOMOVE | SWP_NOSIZE // 标志
);

// 取消置顶
SetWindowPos(
    hwnd,
    HWND_NOTOPMOST,         // 恢复正常层级
    0, 0, 0, 0,
    SWP_NOMOVE | SWP_NOSIZE
);
```

### Dart 代码
```dart
void _setAlwaysOnTop(bool onTop) {
  if (Platform.isWindows) {
    final hwnd = win32.GetForegroundWindow();
    if (hwnd != 0) {
      final hWndInsertAfter = onTop 
          ? win32.HWND_TOPMOST 
          : win32.HWND_NOTOPMOST;
      win32.SetWindowPos(
        hwnd,
        hWndInsertAfter,
        0, 0, 0, 0,
        win32.SWP_NOMOVE | win32.SWP_NOSIZE,
      );
    }
  }
}
```

## 💡 使用技巧

### 技巧 1: 配合小窗口使用
- 窗口尺寸：380 x 700 像素
- 放在屏幕左侧或右侧
- 不会占用太多工作空间

### 技巧 2: 最小化到其他位置
即使启用悬浮窗，仍可最小化到任务栏
- 点击最小化按钮
- 需要时从任务栏恢复

### 技巧 3: 临时禁用
如果需要全屏工作或玩游戏：
1. 暂时关闭悬浮窗模式
2. 完成工作后再开启

### 技巧 4: 日志监控
开启悬浮窗后：
- 实时查看连接日志
- 监控文本接收状态
- 快速诊断问题

## ⚠️ 注意事项

### 1. 性能影响
- 悬浮窗模式几乎不影响性能
- CPU 占用：< 1%
- 内存占用：无额外开销

### 2. 兼容性
- ✅ Windows 10
- ✅ Windows 11
- ✅ Windows 7/8（理论上支持）

### 3. 可能的冲突
某些应用可能会强制自己的窗口置顶：
- 全屏游戏
- 视频播放器
- 系统对话框

在这些情况下，Clip Sync 窗口可能会被短暂遮挡。

### 4. 多显示器
- 窗口会在当前显示器置顶
- 拖动到其他显示器仍保持置顶

## 🔍 故障排除

### 问题 1: 开启后窗口没有置顶
**可能原因**:
- 窗口失去焦点
- 其他应用强制置顶

**解决方法**:
1. 关闭再重新开启开关
2. 点击窗口使其获得焦点
3. 重启应用

### 问题 2: 无法关闭悬浮窗
**解决方法**:
1. 确保窗口可见
2. 点击开关关闭
3. 如果窗口被遮挡，按 Alt+Tab 切换到应用

### 问题 3: 置顶失效
**可能原因**:
- 系统资源管理器重启
- 显示设置更改

**解决方法**:
- 重新开启悬浮窗开关
